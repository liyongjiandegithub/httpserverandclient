int HTTP_ManagedClientRequestEx

( HTTPManagedClientSession* session,
  const HTTP_CHAR* method,
  const HTTP_CHAR* path,
  RTP_TIMESTAMP* ifModifiedSince,
  HTTP_CHAR* contentType,
  HTTP_INT32 contentLength,
  HTTPRequest* request )

Send an HTTP request (low level).

Documentation

HTTP_ManagedClientRequestEx is provided for implementing http based protocols such as SOAP, SSDP and GENA. Use caution when calling this function. Most application needs can be satisfied by using one of the higher level functions HTTP_ManagedClientGet, HTTP_ManagedClientPut, or HTTP_ManagedClientPost.

Parameters:
session - the session over which to perform the GET
method - Commonly GET, POST or PUT. Also supports SOAP and GENA methods SUBSCRIBE, UNSUBSCRIBE, M-SEARCH, M-POST etc.
path - For GET operations the file to get on the server for PUT, the file to PUT
ifModifiedSince - For GET operation instruct the server to send the file only if it has been modified after this date
contentType - the mime type of the data on a POST or PUT ("text/xml", for example)
contentLength - HTTP_CONTENT_LENGTH_NONE for GET operation. For PUT and POST the contentLength is optional. If the contentLength is not specified then the data is sent in chunked transfer-encoding (each call to HTTP_ManagedClientWrite will generate a single chunk). In either case, HTTP_ManagedClientWriteDone must be called once all data has been written.
request - An uninitialized HTTPRequest sized memory buffer for internal use must be provided by the caller
Returns:
0 on success, negative on failure
Preconditions:
a session pointer must have been returned by HTTP_ManagedClientStartTransaction for each time this function is called
